## 数据科学工作不仅仅是编码

很多人认为数据科学家唯一做的工作就是编码和开发机器学习模型。编码技能可能是我们被雇佣的原因，但现实是不同的。在编码之外，我们有许多意想不到的工作要做，只有成为数据科学家后才知道。

这就是为什么在这篇文章中，我想解释我作为数据科学家的经验，我们的工作规范比许多人预期的要大得多。让我们开始吧！

# 1.说服人们

这是什么意思？我们数据科学家需要说服其他人吗？我们要说服什么，为什么它是必要的？。当我告诉你数据科学家需要说服人们时，这种问题可能会出现在你的脑海中——但说服人们是我们的工作。

数据科学、人工智能和机器学习是行业的新人。在过去，我们已经习惯于根据直觉和当时任何有意义的东西来做决定。随着新技术的出现，我们可以处理所有可用的数据来做出数据驱动的决策，而不是仅仅依靠直觉。然而，就像任何新出现的事物一样，许多人对数据科学的前景持怀疑态度。

解释数据科学的前景是数据科学家的工作，我们需要让人们相信我们的工作会让他们的工作受益，让业务受益，让公司受益。**如果他们的老方法已经奏效，他们为什么还要使用你的洞察力和机器学习模型？**这个问题是我们日常工作中常见的一道坎。

以我的经验来看，我遇到过很多商界人士的抵制。不理解我们所做的事情的人，不想改变生活方式的人，害怕新事物的人。我不能责怪他们不理解，因为机器学习模型对于非技术人员来说就像魔术一样。但是，说服我们的工作是数据科学家需要做的事情——如果人们不想使用你的洞察力或模型，这是一种无用的努力。

我是否总能成功说服人们在工作中使用数据产品或机器学习模型？答案是否定的。很多时候，人们会否认数据科学的用处。说服他们的理想方法是什么？

没有说服人的黄金法则，因为每个公司都有自己的文化，每个企业都有自己的方式，每个人都有自己的个性。我能给你的一个建议是:

> 试着用数据展示你的模型的有用性来说服别人，目的是让对方好奇。不要在第一次会面时就立即信服，而是通过“研究”他们的业务问题，尝试说服他们再次会面。

没有充分的证据，你无法让人信服；这就是为什么你需要一些东西来支持你的主张。数据科学家总是会发现不可战胜的人，但你应该总是欣赏更开明的人。

# 2.主要的

起初，数据科学家可能只会致力于开发机器学习模型来解决业务问题，但随着经验的积累，责任也越来越大。随着时间的推移，这种责任将会演变，你将需要领导你工作中的许多事情。

以我的亲身经历为例。公司信任我开发一个机器学习模型来帮助解决我早期的业务问题。对于一个初学者来说，我不太了解业务和数据。然而，有时我会知道我的机器学习模型需要什么来帮助公司——它需要许多部门的组合:销售、营销、数据工程师、it、数字等等。因为我是最知道如何让这个项目成功的人，**我自然会成为项目负责人**，并且需要保持成功的所有变量。

在我们的数据科学工作中，我们有时需要领导，即使我们处于初级职位。范围可能因人而异，但你会以这样或那样的方式经历领导。您可能需要领导项目并与许多人联系，领导您的团队，甚至领导您公司的数据成功方向。

许多数据科学家不喜欢这样做，因为他们认为这不是他们的工作描述。然而，如果你想推进你的事业，让你的模式有所进展，你需要学习领导力。此外，来自您的分析和模型开发的洞察力是有价值的，并且您是最了解它将如何影响业务的人。**如果我们不是引领这场变革的人，还有谁会是呢？**

如果你想获得更多作为数据科学家的领导经验，我给你一些建议:

1.  主动出击。不要只期待事情会为你解决，而是要走自己的路，
2.  不要逃避赋予你的责任。给你的工作是信任的标志，也是展示你领导能力的机会，
3.  向你的老板表达你的愿望。如果你想要体验，你应该讨论一下，
4.  积极参与工作之外的活动。你可能在数据部门工作，但是你的公司里有很多人。尽量多结交一些你平时工作圈子之外的人；会有帮助的。

# 3.商务翻译

数据科学是行业中的新事物，许多人仍然不明白它对业务有多大用处。这种观点意味着数据科学家需要成为数据科学概念和业务人员之间的中间人。

当企业用户带着他们的问题而来，需要我们的帮助时，我总是很兴奋。我为什么兴奋？因为我可以创造性地帮助他们的生意。根据我的个人经验，商务人士有时会有一个很好的主意来解决他们的业务问题，并多少知道该怎么做。尽管如此，他们还不确定如何将其转化为技术执行。这就是我们作为数据科学家来帮助他们分析他们想要什么，并利用我们的创造力将其分解为可行的解决方案的地方。

例如，该公司可能需要分析团队帮助产生更好的销售线索，因为他们当前的流程还没有达到目标。我们可以通过分析根本原因来帮助他们，并开发一个机器学习模型来产生线索。对数据科学家来说听起来很容易，对吗？

然而，想象你自己是数据科学家通过这种方法展示的业务人员:

*“从分析团队来看，我们将分析数据，看看特征之间是否有很强的相关性，我们将开发精度目标为 80%的集成机器学习模型。”*

数据科学家理解上面的术语，但非技术人员不会理解。他们不知道我们所说的技术术语，这些术语只会让他们更加困惑。**困惑的后果通常归结为两件事**:

1.  人们避免使用分析方法，因为他们不理解，
2.  人们接受方法，但执行往往会偏离路线，因为在开始时没有牢固地建立概念。

对于业务人员，我们可以将上面的技术术语转换为:

*“在分析团队中，我们会分析数据，试图找出销售额下降的原因，我们会开发工具来帮助预测有很大机会购买该产品的销售线索。理想情况下，销售线索将贡献收入目标的 10%。”*

上面的句子使用较少的技术术语，并使目标更符合业务目标。

作为数据科学家，我们需要**以人们能够理解的方式将技术解决方案翻译给业务人员**。将数据科学要求完全转化为业务语言需要实践，但并非不可能。我可以提供一些建议来帮助学习翻译技巧:

1.  尽可能避免技术术语，使用商业语言，
2.  不要过度承诺。数据科学家有能力开发模型，但影响成功的变量很多，所以我们的承诺可能会被解释为其他东西，
3.  从商业人士那里挖掘尽可能多的信息。让你的想法与他们一致，如果你不明白就问问题。

# **结论**

雇佣数据科学家是因为他们的分析和编码技能。然而，他们的工作活动不止于此。在本文中，我解释了作为一名数据科学家的三个意想不到的工作活动。它们是:

1.  说服人们
2.  主要的
3.  商务翻译
